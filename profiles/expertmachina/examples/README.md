# ExpertMachina — Examples / Calibration Data

> **NON-CANONICAL. NON-PRODUCTION. NOT EVIDENCE.**
> Nothing here is a governed fact, a ratified contract, or proof of
> anything. In ExpertMachina's own vocabulary: this directory has no
> approval state, no provenance authority, and no ledger presence — it is
> calibration material for the AgenticTeam profile system, full stop.

ExpertMachina is under active development, which makes its material ideal
calibration data and terrible law: shapes change, conventions harden late,
and today's structure is tomorrow's migration. Datasets here validate that
the `expertmachina` profile (globs, templates, proof shapes, workflows)
matches reality — they never define reality.

Standing rules (see `profiles/README.md` for the general policy):

- **Synthetic only.** No production corpus documents, no real vendor or
  customer content, no credentials, no database extracts. Where a real
  development artifact inspired an example, it is reduced to shape
  (structure, field names, conventions) with fabricated content.
- **Not evidence.** No scope contract cites `examples/**`; no proof
  requirement is satisfied by anything here; the profile's standing
  `profiles/**` forbidden glob keeps Builders out mechanically.
- **Promote lessons, not files.** When an example reveals a real rule
  (as the first scoping pass revealed the real forbidden globs), the rule
  moves to `CLAUDE.profile.md` / `scope-template.json` and the example
  remains a specimen.
